大学抢课秒杀系统
一个高性能、高并发、高可用的大学抢课秒杀系统，解决了缓存穿透 / 击穿 / 雪崩、库存超卖、数据一致性等核心问题，提供完整的用户认证、课程管理、抢课退课等功能。
项目简介
本系统针对大学抢课场景的高并发、秒杀特性设计，核心目标是保证在海量请求下系统的稳定性、数据的准确性和用户体验。系统重点解决了秒杀场景下的经典技术难题，并提供了完整的业务功能闭环。
核心技术栈
后端框架：Spring Boot / Spring Cloud（根据实际使用框架修改）
缓存中间件：Redis（缓存、分布式锁、布隆过滤器、JWT 黑名单）
数据库：MySQL（主从 / 读写分离可选）
认证授权：JWT + Redis 黑名单
并发控制：乐观锁、分布式锁
其他：Lombok、MyBatis-Plus、Spring Validation 等
核心功能
1. 用户模块
登录注册：支持用户账号密码登录，生成 JWT 令牌进行身份认证
退出登录：将 JWT 令牌加入 Redis 黑名单，实现即时退登
令牌校验：请求拦截器校验 JWT 有效性，拒绝过期 / 失效令牌访问
2. 课程管理模块
课程 CRUD：支持管理员新增、编辑、删除、查询课程信息
库存管理：设置课程可抢名额，实时更新剩余库存
课程状态：支持课程上架 / 下架、开启 / 关闭抢课功能
3. 抢课 / 退课模块
抢课功能：高并发下的课程秒杀，保证库存准确，无超卖
退课功能：支持用户主动退课，实时恢复课程库存，保证数据一致性
操作校验：防止重复抢课、无库存抢课、越权操作等
4. 缓存优化模块
针对缓存三大核心问题的解决方案：
缓存穿透：布隆过滤器过滤不存在的课程 ID 请求
缓存击穿：分布式锁 + 空值缓存，防止热点 Key 失效后的并发请求打穿数据库
缓存雪崩：缓存 Key 设置随机过期时间，避免大量 Key 同时失效
